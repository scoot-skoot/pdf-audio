Numerical methods are used to solve ODEs when typical exact analytical solutions are difficult (tedious) or impossible to obtain using conventional methods. Instead of producing exact expressions for solutions, these methods approximate the solution by employing discrete value steps to simulate the solution given an initial condition. Euler’s method uses the slope at the current point to advance the simulation forward, but this method is not the best as the error grows the more steps that are taken. This is because we end up truncating part of the Taylor expansion. Runge-Kutta methods improve upon Euler’s method by incorporating additional slope evaluations at each step, which allows for less error due to truncation. The primary reason why the most precise numerical methods are not employed all the time is because there are typically always trade-offs between accuracy and computational costs. For example, while higher-order methods are far more effective at reducing error than lower-order methods, sometimes lower-order methods are used as it is cheaper. Stability is another critical issue when dealing with ODEs. With “stiff” equations, more explicit methods like Euler’s method can become unstable and fail to approximate the solution well unless the step size is very small, even while the true solution is well behaved. It is for this very reason, methods like the Backward Euler’s method was invented as they are able to maintain stability for much larger step sizes.
